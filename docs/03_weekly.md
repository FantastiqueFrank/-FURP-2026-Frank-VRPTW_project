# 第三周实验报告：EVRP-TW 求解器性能评估

**姓名**：赵云龙 
**日期**：2026.7.1 
**实验环境**：OR-Tools 路由求解器，Python 3.13

---

## 1. 实验设置

### 1.1 问题定义
本实验针对带时间窗和电池约束的电动车辆路径问题（EVRP-TW）。  
目标：在满足车辆容量、客户时间窗、电池容量约束的前提下，最小化总行驶距离。  
约束优先级：可行性（所有硬约束）> 目标值。

### 1.2 测试实例
使用 ESOGU 数据集中的两个实例：
- **ESOGU_C20_TW1**：20个客户，1个仓库，10个充电站。
- **ESOGU_C60_TW1**：60个客户，1个仓库，10个充电站。

距离矩阵基于欧几里得距离，时间窗和服务时间来自原始数据。电池消耗率设为 0.2 kWh/km，电池容量为变量（详见结果部分）。

### 1.3 求解器配置
- 求解器：OR-Tools 路由求解器（v9.15.6755）
- 车辆数：20用户为6辆，60用户为10辆
- 车辆容量：350 kg
- 求解策略：
  - 初始解：`PARALLEL_CHEAPEST_INSERTION`
  - 局部搜索：`TABU_SEARCH`
  - 时间限制：120秒（20用户）/ 300秒（60用户）
- 时间窗维度：`slack=500`（20用户）/ `slack=1000`（60用户），`max_time=20000`（20用户）/ `max_time=50000`（60用户）

### 1.4 评估指标
对每个配置记录：
- 可行性（✅/❌）
- 总行驶距离（目标值）
- 运行时间（秒）
- 是否访问充电站（及访问次数）
- 车辆数（固定）

## 3. 讨论

### 3.1 临界容量的规模效应与路线变化
- 20用户临界容量为 697 kWh，但该容量下路线与无电池时完全相同，说明实际消耗远小于697，临界值并不反映真实电量约束。696无解可能源于OR-Tools维度处理中的数值问题（如累积取整或溢出），而非实际电量不足。
- 60用户临界容量 921 kWh 下路线发生改变（新增充电站访问），表明在该规模下电池约束确实影响了路线结构，临界值反映了真实电量瓶颈。

### 3.2 充电站的双重角色
- 在60用户实例中，即使电池容量极大（100000），车辆仍访问充电站 `cs9`，说明充电站同时作为时间调整点。
- 20用户无此现象，可能是因为20用户的时间窗约束较松，不需要充电站作为时间锚点。

### 3.3 电池约束对目标值的影响
- 60用户：从无电池到临界容量，总距离增加 1.25%。
- 20用户：目标值无变化，进一步说明电池约束未实际改变路线。

### 3.4 求解效率
- 无解情况（略低于临界）时，求解器在数秒内返回无解，表明初始解生成阶段即无法构造可行路线。

### 3.5 局限性
- 仅测试了两个实例，且20用户的临界值可能不代表真实物理约束。
- 电池消耗模型为线性简化，未考虑再生制动等因素。
- 充电站建模为“不耗电”而非“充电”，可能低估了充电站的作用。

---

## 4. 结论

本实验通过OR-Tools求解器，评估了电池容量对EVRP-TW可行性和目标值的影响。主要发现如下：

1. **临界容量**：20用户和60用户的临界容量分别为697 kWh和921 kWh，但20用户的临界值未改变路线，可能不反映真实电量约束。
2. **路线变化**：仅在60用户实例中观察到电池约束导致路线改变（增加充电站访问），20用户实例中临界容量下路线与无电池相同。
3. **充电站功能**：在60用户中，充电站同时服务于电量补充和时间调整，即使在电池容量极大时也会被访问。
4. **目标值影响**：在60用户中，电池约束使目标值增加1.25%；20用户无变化。
